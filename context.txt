Crie um quiz interativo em formato de Aplicativo de Teste de Eneagrama Gamificado (Mobile-First)
Título do Projeto: A Jornada do Eneagrama: Teste Interativo Mobile

Objetivo Principal: Desenvolver uma aplicação web de um teste de Eneagrama gamificado, com design mobile-first, para proporcionar uma experiência imersiva e fluida em smartphones. A aplicação deve capturar leads, integrar com um webhook e direcionar os usuários para uma página de vendas após a revelação do resultado.

Princípio Fundamental: Design Mobile-First Toda a aplicação deve ser projetada e desenvolvida priorizando a experiência em dispositivos móveis.
Layout Vertical: O layout deve ser de coluna única, otimizado para visualização e interação em telas verticais.

Elementos de Toque: Botões e opções de resposta devem ser grandes, com espaçamento adequado para evitar toques acidentais.

Tipografia Legível: As fontes devem ser grandes e claras, garantindo conforto na leitura em telas menores.

Performance: O carregamento deve ser rápido em redes móveis. Otimize imagens e use o mínimo de bibliotecas externas.

Responsividade: Após garantir uma experiência perfeita no celular, o design deve se adaptar de forma fluida a tablets e desktops, aproveitando o espaço extra sem quebrar a experiência principal.

Experiência do Usuário (UX) e Gamificação Efeitos Visuais (Foco em Performance Mobile):
Seleção de Resposta: Ao tocar em uma opção, ela deve ter um feedback visual claro (ex: aumentar levemente de tamanho e exibir uma borda colorida). As outras opções ficam semidesvanecidas.

Transições Suaves: Use transições de CSS simples (ex: fade-in, slide-in) entre as perguntas para manter a fluidez sem sobrecarregar o processador do celular.

Barra de Progresso: Uma barra no topo da tela deve indicar o progresso de forma visual e suave.

Efeitos Sonoros (Otimizados para Web Mobile):

Áudios em Base64: Todos os arquivos de áudio devem ser incorporados via strings Base64 para evitar requisições de rede e problemas com bloqueio de autoplay.

Eventos de Som:

Toque na Resposta: Som curto e satisfatório (som_clique.mp3).

Mensagem de Incentivo: Som positivo e sutil (som_incentivo.mp3).

Revelação do Resultado: Som de conquista (som_revelacao.mp3).

Ativação por Toque: A reprodução do áudio deve ser iniciada por uma ação do usuário (o primeiro toque no botão "Começar") para cumprir as políticas dos navegadores mobile.

Fluxo da Aplicação e Estrutura das Telas Tela 1: Boas-Vindas
Layout: Centralizado, com título grande, subtítulo curto e um botão de CTA proeminente que ocupe boa parte da largura da tela.

Conteúdo:

Título: "Inicie sua Jornada de Autoconhecimento."

Subtítulo: "Descubra sua personalidade com este teste interativo."

Botão CTA: "COMEÇAR A JORNADA"

Tela 2: Quiz Interativo

Layout: Uma pergunta por tela. A pergunta no topo, seguida pelas três opções de resposta em botões grandes, um abaixo do outro.

Banco de Perguntas: Utilize o banco de 45 perguntas previamente definido. A lógica deve somar um ponto para o tipo correspondente a cada resposta escolhida.

Mensagens de Incentivo: Modais ou pop-ups sutis que aparecem e desaparecem automaticamente após alguns segundos, sem exigir interação do usuário:

Após a Pergunta 9: "Você está indo muito bem! Continue."

Após a Pergunta 18: "Metade do caminho! Suas respostas estão formando um padrão fascinante."

Após a Pergunta 27: "Falta pouco! Continue com essa honestidade."

Após a Pergunta 36: "Na reta final! Apenas mais algumas perguntas."

Tela 3: Formulário de Captura de Lead

Layout: Otimizado para preenchimento em celular. Labels acima dos campos de input para melhor visualização.

Conteúdo:

Título: "Seu resultado está pronto!"

Subtítulo: "Insira seus dados para revelar seu perfil."

Campos: Nome, Email (usar type="email" para otimizar o teclado do celular).

Botão CTA: "VER MEU RESULTADO AGORA!"

Tela 4: Página de Resultado e Venda

Layout: O resultado do eneatipo deve ser a primeira coisa visível. Abaixo, uma breve descrição e, em seguida, a seção de vendas com um botão de CTA claro e sempre visível.

Ações:

Exibição Dinâmica: "Parabéns, [Nome]! Você é do Tipo [X]: O [Nome do Tipo]."

Descrição: Apresentar o texto correspondente ao eneatipo resultante.

Seção de Vendas: Título, texto persuasivo e o botão CTA final.

Botão CTA de Vendas: "QUERO MEU GUIA COMPLETO" (deve levar à URL da página de vendas).

Especificações Técnicas Frontend:
Usar HTML5, CSS3 e JavaScript puro.

Empregar Flexbox ou CSS Grid para criar os layouts responsivos.

NÃO usar bibliotecas pesadas como jQuery. Foco em performance.

Lógica do Quiz:

Implementar um sistema de pontuação em JavaScript que armazene a contagem para cada um dos 9 tipos.

Ao final, identificar o tipo com a maior pontuação. Em caso de empate, pode-se exibir o primeiro tipo que atingiu a pontuação máxima ou listar os tipos empatados.

Integração (Webhook):

No clique do botão "VER MEU RESULTADO AGORA!" (Tela 3), antes de redirecionar para a tela de resultado, disparar uma requisição fetch do tipo POST para a URL: https://n8n.misterx.com.br/webhook/leads-tati.

Estrutura do JSON a ser enviado:

JSON

"email": "EMAIL_DO_USUARIO",
"eneatipo": "RESULTADO_DO_TESTE"
}
